# Global Claude Instructions

Instructions here apply to all projects unless overridden by a project-level CLAUDE.md.

## Behavior

- Be concise. Skip preamble and trailing summaries — I can read the diff.
- No emojis unless I ask.
- No speculative abstractions or features beyond what was asked.
- No docstrings, comments, or type annotations on code you didn't change.

## Task Tracking

When in a project that uses **beads (`bd`)**, use it for ALL task tracking:
- Do NOT use TodoWrite, TaskCreate, or markdown TODO lists.
- Run `bd prime` for full workflow context.

## Session Completion

Before saying "done", verify:
1. All code changes committed
2. `git push` succeeded — work is not done until pushed

## Architecture & Code

- Trust internal code and framework guarantees; only validate at system boundaries.
- Don't add error handling for scenarios that can't happen.
- Prefer editing existing files over creating new ones.
- Three similar lines of code is better than a premature abstraction.
